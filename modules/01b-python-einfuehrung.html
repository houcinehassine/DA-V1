🐍 Modul 01b · Python Einführung

Was ist Data Science? Warum Python?

Hintergrundwissen zu Python, Data Science, Coding-Prinzipien und empfohlene Literatur.

🔭 Was ist Data Science?

Data Science ist kein reines Programmier-Fach. Um echte Probleme zu lösen, braucht man fundiertes Wissen in drei Bereichen:

  • Programmierung — Code schreiben, Daten verarbeiten, Modelle implementieren
  • Mathematik / Statistik — Algorithmen verstehen, Ergebnisse korrekt interpretieren
  • Domänenwissen — Wissen was die Zahlen bedeuten, richtige Fragen stellen
💡
Um vom BuzzWord-Bingo wegzukommen

Data Science ohne alle drei Bereiche führt zu falschen Schlüssen. Nur wer alle drei kombiniert, kann echten Mehrwert schaffen.

🤔 Why Python?

Python ist die meistgenutzte Programmiersprache im Data Science Bereich — aber warum?

🌍 Sehr populär

Unter den Top-3 Programmiersprachen weltweit. Riesige Community — „StackOverflow your problem".

⚡ Sehr effizient

„Do more in fewer lines of code" — Python ist deutlich kürzer als Java oder C++.

✍️ Clean Code

„Easy to read, debug, extend" — Python-Code liest sich fast wie Englisch.

📦 Riesiges Ökosystem

Zu (fast) jedem Thema gibt es ein Package: NumPy, Pandas, TensorFlow, FastAPI, Django …

🎓 Leicht zu lernen

Ideal als erste Programmiersprache. Einfache Syntax, keine komplizierten Typendeklarationen.

🖥️ Cross-Platform & Open Source

Läuft auf Windows, Mac und Linux gleich. Kostenlos, keine Lizenz nötig.

📅 Python: Ein kurzer Blick zurück

🐣 1989 / 1991 — Entstehung

Guido van Rossum entwickelt Python in den Niederlanden. Ziele: Einfachheit, Lesbarkeit, General Purpose.

🔀 Python 2 vs. Python 3

Lange Zeit existierten zwei Versionen parallel. Python 2.x und Python 3.x waren nicht vollständig kompatibel.

⚰️ 01.01.2020 — Python 2 End of Life

Python 2 wird nicht mehr gepflegt. Wir nutzen ausschließlich Python 3.

🚀 Heute — Standard im Data Science

Python ist die dominierende Sprache für ML, AI und Data Science. Wir nutzen Python 3.x.

Python ist flexibel — nutzbar als Scripting-Sprache, prozedurale Sprache und objektorientierte Sprache. Es ist eine Very-High-Level Programmiersprache.

🧘 Zen of Python

Der „Zen of Python" ist eine Sammlung von 19 Designprinzipien für Python-Programmierung. Du kannst ihn direkt in Python ausgeben:

PYTHON 
import this
Output (Auszug)
Beautiful is better than ugly. Explicit is better than implicit. Simple is better than complex. Readability counts. There should be one obvious way to do it. If the implementation is hard to explain, it's a bad idea.
🌟
Was bedeutet das für uns?

Python-Code soll lesbar, einfach und explizit sein. Vermeide unnötige Komplexität. Ein guter Python-Programmierer schreibt Code, der sich fast wie Englisch liest.

📏 PEP 8 — Style Guide für Python

PEP 8 ist der offizielle Style Guide für Python-Code. Er definiert, wie Code aussehen soll — nicht was er tun soll.

❌ Schlecht (nicht PEP 8)
PYTHON 
x=5
def myFunction( y,z ):
    return y+z
myList=[1,2,3]
✅ Gut (PEP 8 konform)
PYTHON 
x = 5
def my_function(y, z):
    return y + z
my_list = [1, 2, 3]

Die wichtigsten PEP 8 Regeln

Regel Beispiel
Variablen: snake_casestudent_name, nicht studentName
Konstanten: UPPER_CASEMAX_SIZE = 100
Leerzeichen um Operatorenx = 5 + 3, nicht x=5+3
Einrückung: 4 LeerzeichenKein Tab, 4 Spaces
Zeilen max. 79 ZeichenLange Zeilen umbrechen

↔️ Indentation (Einrückung)

Python nutzt Einrückung (Indentation), um die Zugehörigkeit von Code-Zeilen zu bestimmen — anders als die meisten anderen Sprachen, die geschweifte Klammern {} verwenden.

❌ Falsche Einrückung
PYTHON 
def my_function():
return 42   # IndentationError!
✅ Korrekte Einrückung
PYTHON 
def my_function():
    return 42  # 4 Spaces!
💡
Warum ist das gut?

Indentation zwingt Programmierer zu sauberem, lesbarem Code. In Python sieht man sofort, was zu einer Funktion, Schleife oder Bedingung gehört — ohne Klammern zählen zu müssen.

PYTHON 
# Die Einrückung zeigt, welcher Code "innerhalb" der if-Bedingung liegt:
x = 10

if x > 5:
    print("x ist größer als 5")   # ← gehört zum if-Block
    print("das gilt noch")         # ← auch im if-Block

print("das gilt immer")            # ← außerhalb des if-Blocks
Output
x ist größer als 5 das gilt noch das gilt immer
