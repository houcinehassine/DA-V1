Data Preparation & Feature Engineering

Rohe Daten in modellfertige Features umwandeln: Encoding, Feature Extraction, Selection und Scaling — der Schlüssel zu guten ML-Modellen.

Inhalte
→ Label- & One-Hot-Encoding → Feature Hashing → Bag of Words → Feature Selection → Feature Scaling → Quantisierung
💡
Warum Data Preparation?

ML-Algorithmen verstehen nur Zahlen. Kategorische Daten müssen encodiert, Texte vektorisiert und alle Features auf vergleichbare Skalen gebracht werden. "Garbage in → Garbage out."

🏷️ Feature Extraction: Encoding

Kategorische Variablen (z.B. 'apple', 'banana') müssen in Zahlen umgewandelt werden, damit ML-Modelle damit rechnen können.

Label-Encoding

Jede Kategorie bekommt eine eindeutige ganze Zahl. Gut für ordinale Daten (z.B. klein=0, mittel=1, groß=2).

PYTHON
from sklearn.preprocessing import LabelEncoder
import numpy as np

categories = np.array(['apple', 'apple', 'milk', 'banana', 'coffee'])

le = LabelEncoder()
label_encoded = le.fit_transform(categories)
print(label_encoded)   # → [0 0 3 1 2]
print(le.classes_)     # → ['apple' 'banana' 'coffee' 'milk']
Output
[0 0 3 1 2]
⚠️
Problem mit Label-Encoding bei nominalen Daten

Das Modell könnte annehmen: banana (1) < coffee (2) — das ist falsch bei gleichwertigen Kategorien! Lösung: One-Hot-Encoding.

One-Hot-Encoding

Jede Kategorie bekommt eine eigene Spalte (0 oder 1). Für nominale Daten ohne Reihenfolge.

PYTHON
from sklearn.preprocessing import OneHotEncoder

categories = np.array(['apple', 'milk', 'banana', 'coffee']).reshape(-1, 1)

ohe = OneHotEncoder(sparse_output=False)
one_hot = ohe.fit_transform(categories)
print(one_hot)

# apple  banana  coffee  milk
# [1,    0,      0,      0]   ← apple
# [0,    0,      0,      1]   ← milk
# [0,    1,      0,      0]   ← banana
# [0,    0,      1,      0]   ← coffee

# Mit Pandas (einfacher):
pd.get_dummies(df['kategorie_spalte'])

🔨 Feature Hashing

Bei sehr vielen Kategorien (z.B. Millionen von Webseiten) wird One-Hot-Encoding zu groß. Feature Hashing (Hashing Trick) bildet Kategorien auf einen festen Vektorraum ab — ohne alle Kategorien zu kennen.

PYTHON
from sklearn.feature_extraction import FeatureHasher

data = [{'animal': 'dog'}, {'animal': 'cat'}, {'animal': 'horse'}]

hasher = FeatureHasher(n_features=8, input_type='dict')
hashed = hasher.transform(data).toarray()
print(hashed.shape)   # → (3, 8)  — 3 Samples, 8 Features

📝 Bag of Words (Text → Zahlen)

Texte müssen in Zahlenvektoren umgewandelt werden. Bag of Words zählt, wie oft jedes Wort in einem Text vorkommt.

PYTHON
from sklearn.feature_extraction.text import CountVectorizer

corpus = [
    'Data Science is great',
    'Python is great for data',
    'Machine learning and data science'
]

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(corpus)

print(vectorizer.get_feature_names_out())
# ['and', 'data', 'for', 'great', 'is', 'learning', 'machine', 'python', 'science']

print(X.toarray())
# [[0, 1, 0, 1, 1, 0, 0, 0, 1],  ← "Data Science is great"
#  [0, 1, 1, 1, 1, 0, 0, 1, 0],  ← "Python is great for data"
#  [1, 1, 0, 0, 0, 1, 1, 0, 1]]  ← "Machine learning and data science"

🎯 Feature Selection

Nicht alle Features sind nützlich — irrelevante Features können ein Modell sogar verschlechtern. Feature Selection wählt die relevantesten aus.

Einfache Feature Selection (manuelle Auswahl)

PYTHON
# Nur bestimmte Spalten behalten
X = df[['feature_1', 'feature_3', 'feature_5']]

# Korrelation mit Zielvariable — wenig korrelierte Features entfernen
corr = df.corr()['target'].abs().sort_values(ascending=False)
print(corr)

SelectKBest — automatisch die besten k Features

PYTHON
from sklearn.feature_selection import SelectKBest, f_classif

# Die besten 5 Features für Klassifikation
selector = SelectKBest(score_func=f_classif, k=5)
X_selected = selector.fit_transform(X, y)

print(X_selected.shape)           # → (n_samples, 5)
print(selector.get_support())     # → [True, False, True, ...]  welche Features

⚖️ Feature Scaling

Viele ML-Algorithmen (z.B. kNN, SVM, Gradient Descent) sind empfindlich gegenüber unterschiedlichen Skalen. Alter (0–100) und Gehalt (0–100.000) müssen normiert werden.

MinMax-Transformation (Normalisierung)

Skaliert alle Werte auf den Bereich [0, 1].

PYTHON
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()
X_scaled = scaler.fit_transform(X)

# Formel: x_neu = (x - x_min) / (x_max - x_min)
# Ergebnis: alle Werte zwischen 0 und 1

Standardisierung (Z-Score Normalisierung)

Transformiert auf Mittelwert = 0 und Standardabweichung = 1. Robuster bei Ausreißern.

PYTHON
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Formel: x_neu = (x - mean) / std
# Ergebnis: Mittelwert ≈ 0, Std ≈ 1

Quantisierung — Zahlen in Klassen einteilen

PYTHON
import pandas as pd

# Feste Klassengrößen
pd.cut(df['alter'], bins=[0, 18, 35, 60, 100],
       labels=['Kind', 'Jung', 'Mittel', 'Senior'])

# Adaptive Klassengrößen (gleich viele Elemente je Klasse)
pd.qcut(df['alter'], q=4,
        labels=['Q1', 'Q2', 'Q3', 'Q4'])
Methode Ergebnis Wann nutzen?

💡
fit_transform vs. transform

Wichtig: fit_transform(X_train) auf Trainingsdaten anwenden — das lernt die Parameter (min, max, mean, std). Auf Testdaten nur transform(X_test) — sonst "schummelt" man mit Testdaten-Wissen!

🎯 Selbsttest

📋 Teste dein Wissen — Data Preparation
1. Wann ist One-Hot-Encoding besser als Label-Encoding?
Immer — One-Hot ist immer besser
Bei nominalen Kategorien ohne Reihenfolge (z.B. Farben, Länder) — Label-Encoding würde falsche Ordnung suggerieren
Bei sehr vielen Kategorien (über 1000)
2. Was macht StandardScaler?
Skaliert alle Werte auf [0, 1]
Transformiert auf Mittelwert = 0 und Standardabweichung = 1 (Z-Score)
Entfernt Ausreißer aus den Daten
3. Warum soll man auf Testdaten nur .transform() und nicht .fit_transform() aufrufen?
Weil transform() schneller ist
Damit keine Information aus den Testdaten in das Scaling-Modell fließt (Data Leakage vermeiden)
fit_transform() funktioniert nur auf Trainingsdaten technisch
4. Was ist Bag of Words?
Eine Methode zum Encodieren von Kategorien in Zahlen
Eine Methode, die Texte in Wortzähl-Vektoren umwandelt
Ein Feature-Scaling-Verfahren für Textdaten
