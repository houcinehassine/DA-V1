Funktionen & Packages

💡 Warum Funktionen?

𝑦 = 𝑓(𝑥)
Mathematisch:
x = Eingabewert, f = Funktion, y = Ausgabewert
In Python:
input → function → output
result = my_function(argument)
⭐
Store and Reuse — einmal schreiben, immer wieder nutzen

"Wenn man öfter denselben Code schreibt → Funktion."
"Sobald du etwas mehr als drei mal schreiben musst → mache eine Funktion daraus."

🔧 Funktionen definieren — def

Mit dem Schlüsselwort def definiert man eine eigene Funktion. Funktionsaufrufe nennt man auch Calls oder Invokes — sie werden durch runde Klammern () bewirkt.

PYTHON
def greet_user(user_name):
    """Diese Funktion begrüßt den Nutzer."""
    print(f"Hallo {user_name.title()}, wie geht es Dir?")

greet_user("markus")   # ← dies ist ein "Call" (Funktionsaufruf)
Output
Hallo Markus, wie geht es Dir?
def
Schlüsselwort: "definiere eine Funktion"
Parameter
Eingabewerte in den Klammern
Docstring
Beschreibung in """...""" — guter Stil!

Professioneller Docstring-Stil

PYTHON
def some_function(input_argument_1, input_argument_2):
    """
    Hier beschreibt man kurz den Sinn und Zweck der Funktion.
    :param input_argument_1: Was dieser Parameter bedeutet.
    :param input_argument_2: Was dieser Parameter bedeutet.
    :return: Was die Funktion zurückgibt.
    """
    return_value = input_argument_1 ** input_argument_2
    return return_value

result = some_function(2, 3)
print(result)   # → 8

📋 Die 4 Möglichkeiten von Funktionen

Funktionen können mit oder ohne Eingabe- und Ausgabeparameter definiert werden. Alle 4 Kombinationen sind gültig:

① Mit Ein- UND Ausgabeargumenten
PYTHON
def square(x):
    return x ** 2

result = square(5)   # → 25
② Mit Eingabe, OHNE Ausgabe (kein return)
PYTHON
def greet(name):
    print(f"Hallo {name}!")
    # kein return → gibt None zurück

greet("Max")   # → Hallo Max!
③ OHNE Eingabe, MIT Ausgabe
PYTHON
import numpy as np

def simulate_dice():
    return np.random.randint(1, 7)

wurf = simulate_dice()   # → z.B. 4
④ OHNE Ein- UND Ausgabeargumente
PYTHON
def hello_world():
    print("Hello world!")

hello_world()   # → Hello world!

📌 Argumenttypen

1. Positional Arguments — Reihenfolge zählt

PYTHON
def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type}.")
    print(f"My {animal_type}'s name is {pet_name}.")

describe_pet('hamster', 'Larry')   # ✅ Richtig
describe_pet('Larry', 'hamster')   # ⚠️ Falsche Reihenfolge!
Output (1. Aufruf)
I have a hamster. My hamster's name is Larry.

2. Keyword Arguments — Name egal, Position egal

PYTHON
describe_pet(animal_type='fish', pet_name='Nemo')
describe_pet(pet_name='Nemo', animal_type='fish')   # gleicher Output

3. Default Values — optionale Parameter

PYTHON
def describe_pet(pet_name, animal_type='dog'):   # 'dog' ist der Standard
    print(f"I have a {animal_type}.")
    print(f"My {animal_type}'s name is {pet_name}.")

describe_pet('Larry')             # → I have a dog. My dog's name is Larry.
describe_pet('Larry', 'hamster')  # → I have a hamster. ...
💡
Reihenfolge der Parameter

Parameter ohne Default-Wert müssen vor Parametern mit Default-Wert stehen: def f(a, b, c=1, d=2)
Aufruf f(10, c, d) mit c=11, d=12: a=10, b=11, c=12, d=2 → gibt 10 11 12 2 aus.

↩️ return & mehrere Rückgabewerte

PYTHON
def some_descriptive_statistics(values):
    """Berechnet Mittelwert und Varianz einer Werteliste."""
    mean = sum(values) / len(values)
    variance = sum((v - mean) ** 2 for v in values) / len(values)
    return mean, variance   # gibt ZWEI Werte zurück (als Tuple)

# Mehrere Rückgabewerte entpacken:
mean, var = some_descriptive_statistics([1, 3, 5, 3, 2, 5, 7, 8])
print(f"Mittelwert: {mean}")
print(f"Varianz:    {var}")
Output
Mittelwert: 4.25 Varianz: 4.1875
🧘 Coding Yoga: Funktionen I — verschachtelte Aufrufe Aufgabe ▶

Was gibt print(magic(magic(1, 2), magic(2, 2))) aus?

PYTHON
def magic(x, y):
    return x ** y + 1

print(magic(magic(1, 2), magic(2, 2)))
LÖSUNG
# magic(1, 2) = 1**2 + 1 = 2
# magic(2, 2) = 2**2 + 1 = 5
# magic(2, 5) = 2**5 + 1 = 33
# Ergebnis: 33
🧘 Coding Yoga: Funktionen II — while + pop() Fortgeschritten ▶

Was gibt print(crazy([1, 2, 3, 4, 5], False)) aus?

PYTHON
def crazy(some_list, true):
    while true - 1:
        some_list.pop()
        if len(some_list) == 2:
            true = True
    return sum(some_list)

print(crazy([1, 2, 3, 4, 5], False))
LÖSUNG
# true=False → False-1 = -1 (truthy) → Schleife läuft
# pop() → [1,2,3,4], len=4 ≠ 2 → weiter
# pop() → [1,2,3],   len=3 ≠ 2 → weiter
# pop() → [1,2],     len=2 == 2 → true = True
# while True-1 = 0 (falsy) → STOP
# return sum([1, 2]) = 3
# Ergebnis: 3
🧘 Coding Yoga: Funktionen III — Funktionen als Werte Fortgeschritten ▶

Was gibt print(a + b + c) aus?

PYTHON
def add(a, b): return a + b
def mult(a, b): return a * b

func_dict = {0: add, 1: mult}   # Funktionen als Dictionary-Werte!

a = func_dict[0](1, 2)
b = func_dict[1](2, 3)
c = func_dict[0](a, b)
print(a + b + c)
LÖSUNG
# func_dict[0] = add, func_dict[1] = mult
# a = add(1, 2) = 3
# b = mult(2, 3) = 6
# c = add(3, 6) = 9
# a + b + c = 3 + 6 + 9 = 18
# Ergebnis: 18
#
# Wichtig: In Python sind Funktionen "First-Class Objects" —
# sie können in Listen, Dictionaries und als Argumente übergeben werden!

🏗️ Top-Down Prinzip & Code-Organisation

📐
Top-Down Prinzip

Wenn Code zu lang oder zu komplex wird: in kleinste, logische Teile herunterbrechen und für jeden Teil eine Funktion schreiben. Strukturiere deinen Code in Paragraphen — dabei helfen Kommentare und Funktionen.

  • Wenn man öfter denselben Code schreibt → Funktion
  • Sobald etwas mehr als 3 Mal geschrieben wird → Funktion daraus machen
  • Lange Funktionen (>20 Zeilen) → in kleinere Funktionen aufteilen
  • Viele Funktionen → in Modules (= .py-Dateien) organisieren
  • Viele Modules → in Packages (= Ordner) organisieren

⚡ Lambda-Funktionen

Eine Lambda-Funktion ist eine anonyme (namenlose) Einzeilen-Funktion. Lambda-Funktionen werden normalerweise als Argument für Higher-Order Functions (wie filter(), map(), sorted()) verwendet.

Normale Funktion
PYTHON
def quad(x):
    return x ** 2

quad(5)   # → 25
⭐ Lambda
PYTHON
quad = lambda x: x ** 2

quad(5)   # → 25
📐
Syntax: lambda parameter: ausdruck

Kein def, kein return, kein Funktionsname nötig. Lambda gibt den Ausdruck automatisch zurück. Für mehrere Parameter: lambda x, y: x + y

🔍 filter() & map()

Diese eingebauten Higher-Order Functions arbeiten perfekt mit Lambdas zusammen, um Listen zu verarbeiten.

PYTHON
my_list = [1, 5, 4, 6, 8, 11, 3, 12]

# filter(): Behalte nur Elemente, für die die Funktion True zurückgibt
even = list(filter(lambda x: x % 2 == 0, my_list))
print(even)    # → [4, 6, 8, 12]

# map(): Wende Funktion auf jedes Element an → neue Liste
doubled = list(map(lambda x: x * 2, my_list))
print(doubled) # → [2, 10, 8, 12, 16, 22, 6, 24]
🧘 Coding Yoga: Lambda + filter + map Aufgabe ▶

Was gibt print(sick) aus?

PYTHON
i_feel_cold = [100, 112, 114]
sick = list(filter(lambda x: x > 39,
                   map(lambda x: x / 2 - 16, i_feel_cold)))
print(sick)
LÖSUNG
# map(lambda x: x/2 - 16, [100, 112, 114])
#   → [34.0, 40.0, 41.0]   (ungefähre Fahrenheit→Celsius Formel)
# filter(lambda x: x > 39, [34.0, 40.0, 41.0])
#   → [40.0, 41.0]   (Fieber!)
# Ergebnis: [40.0, 41.0]
🧘 Coding Yoga: Lambda in einer Liste Fortgeschritten ▶

Was gibt print(sum(my_list)) aus?

PYTHON
weird_list = [
    lambda x: (x % 2 == 0),
    lambda x: x % 3
]
not_so_weird_list = [1, 2, 3, 4, 5]
my_list = []
for strange in weird_list:
    for item in filter(strange, not_so_weird_list):
        my_list.append(item)
print(sum(my_list))
LÖSUNG
# Lambda 1: x % 2 == 0  → gerade Zahlen
#   filter über [1,2,3,4,5] → [2, 4]  → my_list += [2, 4]
# Lambda 2: x % 3       → Zahlen bei denen Rest ≠ 0 (truthy)
#   filter über [1,2,3,4,5] → [1, 2, 4, 5]  → my_list += [1,2,4,5]
# my_list = [2, 4, 1, 2, 4, 5]
# sum([2, 4, 1, 2, 4, 5]) = 18
# Ergebnis: 18

📦 Module & Packages

Definitionen

Module — jede Python-Datei ist ein Modul:

"Any Python file is a module, its name being the file's base name without the .py extension."

Package — ein Ordner mit Modulen:

"A package is a collection of Python modules: while a module is a single Python file, a package is a directory of Python modules containing an additional __init__.py file."

Funktionen
viele Funktionen → in .py-Datei
Modul (.py)
viele Module → in Ordner
Package (Ordner)
+ __init__.py
📁
Module-Pfade: Punkte statt Slashes

Module entsprechen der Ordnerstruktur — nur mit Punkten statt Slashes:
numpy.random.randint = numpy/random/randint
import numpy.random as rand → importiert das Untermodul random aus dem Package numpy

4 Import-Stile

PYTHON
# Stil 1: Ganzes Paket importieren (langer Name)
import numpy
numpy.random.randint(1, 6)

# Stil 2: Mit Alias ⭐ (empfohlen — Standard in Data Science)
import numpy as np
import pandas as pd
np.random.randint(1, 6)

# Stil 3: Nur bestimmte Funktion importieren
from numpy import random
random.randint(1, 6)

# Stil 4: Paket-Untermodul mit Alias
import numpy.random as rand
rand.randint(1, 6)
⭐
Standard-Aliases in Data Science
