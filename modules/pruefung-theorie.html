❓ Theoriefragen Klick zum Aufklappen

Expression hat einen Rückgabewert (kann einem Ausdruck zugewiesen werden), z.B. 2 + 2, len([1,2,3]), "abc".upper().

Statement führt eine Aktion aus, hat keinen Rückgabewert, z.B. x = 5, print("hallo"), for i in range(5):.

= ist der Zuweisungsoperator (Assignment): weist einen Wert einer Variablen zu. x = 5

== ist der Vergleichsoperator (Comparison): vergleicht zwei Werte und gibt True oder False zurück. x == 5 → True

Der Python-Parser würde den Beginn als Zahl interpretieren und einen SyntaxError auslösen. Beispiel: 1abc = 5 → SyntaxError.

Erlaubt: abc1, var_1, _x. Nicht erlaubt: 1abc, 3_vars.

Alle folgenden Werte ergeben bool(x) == False:

  • 0 und 0.0 (Zahlen)
  • "" (leerer String)
  • [] (leere Liste), {} (leeres Dict), () (leeres Tuple)
  • None

Alle anderen Werte sind Truthy: 1, "text", [0] (Liste mit einem Element!).

Mutable (veränderbar): list, dict, set – können nach der Erstellung verändert werden.

Immutable (unveränderbar): int, float, str, tuple – können nicht verändert werden. Strings haben kein .append()!

Wichtig: s = s + "x" erstellt einen neuen String, verändert s nicht in-place.

.append(x) fügt x als einzelnes Element an:
[1,2].append([3,4]) → [1, 2, [3, 4]]

.extend([a,b]) fügt jedes Element der Liste einzeln an:
[1,2].extend([3,4]) → [1, 2, 3, 4]

for: wenn die Anzahl der Iterationen bekannt ist oder über eine Sequenz (Liste, range, Dict) iteriert wird.

while: wenn eine Bedingung geprüft werden muss und man nicht weiß, wie viele Iterationen nötig sind (z.B. Benutzereingabe, Zahlen raten). Achtung: immer Abbruchbedingung sicherstellen!

dict["key"]: gibt den Wert zurück, wirft aber KeyError wenn der Key nicht existiert.

dict.get("key", default): gibt den Wert zurück, oder default wenn der Key fehlt – ohne Fehler. Sehr nützlich bei unbekannten Keys.

A * B: Elementweise Multiplikation (beide Arrays brauchen gleiche Form). Element für Element multipliziert.

np.dot(A, B): Matrix-Multiplikation (echtes Skalarprodukt / Matrixprodukt). Für neuronale Netze: np.dot(X, W) + b

Bei 1D: np.dot ist das Skalarprodukt (summiert die Produkte). Bei 2D: echte Matrizenmultiplikation.

data hat typischerweise die Form (n_samples, n_features) – also Zeilen = Datenpunkte, Spalten = Features.

np.corrcoef erwartet die Form (n_features, n_samples) – jede Zeile ist ein Feature. Das Transponieren mit .T dreht die Achsen: aus (100, 5) wird (5, 100). Das Ergebnis ist eine (5, 5) Korrelationsmatrix.

Der p-Wert gibt die Wahrscheinlichkeit an, den beobachteten Unterschied zufällig zu sehen, wenn H₀ wahr wäre.

  • p < 0.05: Ergebnis ist signifikant. H₀ ablehnen. Der Unterschied ist mit 95% Sicherheit real (nicht zufällig).
  • p ≥ 0.05: Nicht signifikant. H₀ beibehalten. Kein Beweis für einen Unterschied.

H₀ (Nullhypothese): kein Unterschied (µ₁ = µ₂). H₁: Unterschied vorhanden.

R² misst den Anteil der Varianz der Zielvariable, der durch das Modell erklärt wird.

  • R² = 1.0: perfekte Vorhersage
  • R² = 0.0: Modell erklärt nichts (schlechter als Mittelwert)
  • R² < 0: Modell ist schlechter als simple Mittelwertschätzung

Berechnung: model.score(X_test, y_test)

Overfitting = das Modell hat sich zu stark an die Trainingsdaten angepasst und kann nicht verallgemeinern.

Erkennung: Train R² ist viel höher als Test R² (z.B. Train = 0.99, Test = 0.30)

Ursache: Polynomgrad zu hoch, Modell zu komplex

Gegenteil – Underfitting: Train R² und Test R² beide niedrig → Modell zu einfach (Grad zu niedrig)

Lösung: niedrigeren Grad wählen, mehr Trainingsdaten, Cross-Validation

Teilt Daten in Trainings- und Testdaten auf (typisch 80% / 20%).

Das Modell wird nur auf den Trainingsdaten trainiert. Die Testdaten bleiben "unsichtbar" und dienen als realistische Bewertungsgrundlage.

Ohne Split: Modell würde auf denselben Daten bewertet, auf denen es trainiert wurde → überschätzte R²-Werte, Overfitting wird nicht erkannt.

random_state=42: macht den Split reproduzierbar (gleiche Aufteilung bei jedem Aufruf).

Cross-Validation (k-Fold): Das Dataset wird in k gleich große Teile (Folds) aufgeteilt. Das Modell wird k-mal trainiert – jedes Mal auf k-1 Folds, bewertet auf dem verbleibenden Fold.

Vorteil: jeder Datenpunkt wurde einmal als Testdaten genutzt → stabilere, repräsentativere Bewertung. Weniger abhängig vom zufälligen Split.

Ergebnis: Array von k Scores → scores.mean() und scores.std() zeigen Durchschnitt und Variabilität.

sklearn modelliert X als Feature-Matrix mit Shape (n_samples, n_features). Jede Zeile = ein Datenpunkt, jede Spalte = ein Feature.

Bei nur einem Feature: x.reshape(-1, 1) – das -1 lässt Python die Zeilenanzahl berechnen, die 1 ist die Spaltenanzahl.

Beispiel: np.array([1,2,3]).reshape(-1,1) → [[1],[2],[3]] mit Shape (3, 1)
