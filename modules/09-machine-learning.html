Einführung in Machine Learning

Der sklearn-Prozess, Lineare Regression als erstes Modell, Polynomiale Regression, und die wichtige Unterscheidung zwischen Overfitting und Underfitting.

Inhalte

🤖 Was ist Machine Learning?

Machine Learning (ML) ist ein Teilgebiet der KI, bei dem Algorithmen aus Daten lernen, ohne explizit programmiert zu werden. Statt Regeln vorzugeben, findet das Modell selbst Muster.

🏷️
Supervised Learning
Lernt aus gelabelten Daten (X→y). Regression & Klassifikation.
🔍
Unsupervised Learning
Findet Muster ohne Labels. Clustering, Dimensionsreduktion.
🎮
Reinforcement Learning
Lernt durch Belohnung & Bestrafung in einer Umgebung.
📌
In diesem Kurs: Supervised Learning

Wir haben ein Feature-Matrix X (Eingabe) und einen Zielvektor y (gewünschte Ausgabe). Das Modell lernt die Funktion f: X → y.

⚙️ Der sklearn-Prozess: fit → predict → score

Scikit-learn (sklearn) hat eine einheitliche API für alle Modelle — immer die gleichen 3 Schritte:

1️⃣
model.fit(X_train, y_train)
Modell trainieren:
Parameter aus Daten lernen
→
2️⃣
model.predict(X_test)
Vorhersagen treffen:
y für neue X berechnen
→
3️⃣
model.score(X_test, y_test)
Modell bewerten:
Wie gut sind die Vorhersagen?

✂️ Train / Test Split

Bevor man trainiert, teilt man die Daten in Trainingsdaten (Modell lernt) und Testdaten (Modell wird bewertet). So sieht man, ob das Modell auch auf unbekannten Daten funktioniert.

PYTHON
from sklearn.model_selection import train_test_split

# 80% Training, 20% Test
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,      # 20% für Test
    random_state=42     # reproduzierbar
)

print(f"Training:  {X_train.shape}")   # → (80, n_features)
print(f"Test:      {X_test.shape}")    # → (20, n_features)

📈 Lineare Regression

Das einfachste Regressions-Modell: Es findet die beste Gerade durch die Datenpunkte — y = m·x + b.

PYTHON
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.datasets import make_regression
import numpy as np

# Daten erstellen
X, y = make_regression(n_samples=100, n_features=1,
                        noise=10, random_state=42)

# Train/Test Split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 1. Modell erstellen & trainieren
model = LinearRegression()
model.fit(X_train, y_train)

# 2. Vorhersagen
y_pred = model.predict(X_test)

# 3. Bewerten (R² Score: 1.0 = perfekt, 0 = schlecht)
score = model.score(X_test, y_test)
print(f"R² = {score:.3f}")

# Koeffizienten
print(f"Steigung (m): {model.coef_[0]:.2f}")
print(f"Achsenabschnitt (b): {model.intercept_:.2f}")
📐 Modell-Formel
ŷ = m · x + b
m = model.coef_ (Steigung)
b = model.intercept_ (Achsenabschnitt)
📊 R² Score
• R² = 1.0 → perfekte Vorhersage
• R² = 0.0 → Modell sagt immer Mittelwert
• R² < 0 → schlechter als Mittelwert

🌊 Polynomiale Regression

Wenn die Daten einem Kurvenverlauf folgen (nicht linear), nutzen wir Polynomiale Regression: Wir erstellen neue Features wie x², x³, … und nutzen dann weiterhin lineare Regression.

PYTHON
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

# Grad 2: x → [1, x, x²]
# Grad 3: x → [1, x, x², x³]

# Mit Pipeline (empfohlen)
poly_model = Pipeline([
    ('poly',  PolynomialFeatures(degree=2)),
    ('linear', LinearRegression())
])

poly_model.fit(X_train, y_train)
score = poly_model.score(X_test, y_test)
print(f"R² (poly, degree=2): {score:.3f}")
📐
Was macht PolynomialFeatures?

Aus einem Feature x wird ein erweitertes Feature-Set: [1, x, x²] (degree=2) oder [1, x, x², x³] (degree=3). Dann trainiert LinearRegression auf diesen erweiterten Features — das Ergebnis ist eine Kurve, nicht mehr eine Gerade.

⚖️ Overfitting vs. Underfitting

Das zentrale Problem beim Machine Learning: Das Modell soll gut auf neuen, unbekannten Daten funktionieren — nicht nur auf den Trainingsdaten.

📉
Underfitting
Modell zu einfach — lernt nicht genug aus den Daten
Train Score: niedrig
Test Score: niedrig

Lösung: komplexeres Modell
✅
Gutes Modell
Generalisiert gut auf neue Daten
Train Score: hoch
Test Score: hoch

Ziel!
📈
Overfitting
Modell zu komplex — lernt "auswendig", nicht die Muster
Train Score: sehr hoch
Test Score: niedrig

Lösung: einfacheres Modell, mehr Daten
PYTHON
# Overfitting erkennen: Train Score viel höher als Test Score
for degree in [1, 2, 5, 10, 15]:
    model = Pipeline([
        ('poly', PolynomialFeatures(degree=degree)),
        ('lin',  LinearRegression())
    ])
    model.fit(X_train, y_train)
    train_score = model.score(X_train, y_train)
    test_score  = model.score(X_test,  y_test)
    print(f"degree={degree:2d}  Train R²={train_score:.3f}  Test R²={test_score:.3f}")

# Typische Ausgabe:
# degree= 1  Train R²=0.612  Test R²=0.598   ← Underfitting
# degree= 2  Train R²=0.831  Test R²=0.819   ← Gut
# degree= 5  Train R²=0.921  Test R²=0.875   ← Gut
# degree=10  Train R²=0.987  Test R²=0.412   ← Overfitting!
# degree=15  Train R²=0.999  Test R²=-2.341  ← Starkes Overfitting!
🧘 Coding Yoga: Vollständiger sklearn-Workflow Aufgabe ▶

Bringe die folgenden Schritte in die richtige Reihenfolge für einen vollständigen ML-Workflow:

A) model.score(X_test, y_test)
B) train_test_split(X, y)
C) model.fit(X_train, y_train)
D) Daten laden & vorbereiten
E) model.predict(X_test)
F) LinearRegression()
✅
Richtige Reihenfolge: D → B → F → C → E → A

1. Daten laden (D) → 2. Split (B) → 3. Modell erstellen (F) → 4. Trainieren (C) → 5. Vorhersagen (E) → 6. Bewerten (A)

🎯 Selbsttest

📋 Teste dein Wissen — Machine Learning
1. Was bedeutet R² = 0.95?
Das Modell macht 95% Fehler
Das Modell erklärt 95% der Varianz in den Daten — sehr gute Vorhersage
95% der Trainingsdaten wurden richtig klassifiziert
2. Ein Modell hat Train R² = 0.99 aber Test R² = 0.45. Was ist das Problem?
Underfitting — das Modell ist zu einfach
Overfitting — das Modell hat die Trainingsdaten auswendig gelernt statt verallgemeinert
Der Train/Test Split war falsch
3. Was macht PolynomialFeatures(degree=2) mit einem Feature x?
Es berechnet x²
Es erstellt die Features [1, x, x²] — das ermöglicht Kurvenanpassung
Es normalisiert x auf den Bereich [0, 2]
4. Warum teilt man Daten in Training und Test?
Um Rechenzeit zu sparen
Weil sklearn das so verlangt
Um zu prüfen, ob das Modell auf unbekannten Daten generalisiert — nicht nur die Trainingsdaten "auswendig" lernt
