A1

Tage konvertieren

int · float · arithmetic

Schreibe ein Programm, das eine gegebene Anzahl von Tagen in Jahre, Wochen und übrig bleibende Tage umrechnet.

Beispiel: 1000 Tage = 2 Jahre, 38 Wochen, 4 Tage

AUFGABE
# Starter Code
days = 1000

# Deine Lösung hier
✅ Musterlösung
LÖSUNG
days = 1000

# Ganzzahlige Division (//) gibt uns die vollen Jahre ohne Rest
years = days // 365

# Mit % (Modulo) bekommen wir den Rest nach Abzug der Jahre
remaining = days % 365

# Aus dem Rest berechnen wir volle Wochen
weeks = remaining // 7

# Und nochmal Modulo für die übrig bleibenden einzelnen Tage
remaining_days = remaining % 7

print(f"{days} Tage = {years} Jahre, {weeks} Wochen, {remaining_days} Tage")
A2

Dreisatz

float · variables

Ein Auto fährt 120 km in 1,5 Stunden. Berechne, wie weit es in 2,75 Stunden kommt und wie lange es für 200 km braucht. Verwende sprechende Variablennamen.

AUFGABE
# Starter Code
distance_km = 120
time_h = 1.5

# Deine Lösung hier
✅ Musterlösung
LÖSUNG
distance_km = 120
time_hours = 1.5

# Geschwindigkeit = Weg / Zeit — das ist der Kernansatz des Dreisatzes
speed = distance_km / time_hours
print(f"Geschwindigkeit: {speed:.1f} km/h")

# Mit der Geschwindigkeit können wir beliebige Zeit→Distanz Fragen lösen
print(f"In 2.75 Stunden: {speed * 2.75:.1f} km")

# Und Distanz→Zeit: Zeit = Weg / Geschwindigkeit
print(f"Für 200 km: {200 / speed:.2f} Stunden")
A3

Geburtstagsparadoxon

float · probability · math

Berechne die Wahrscheinlichkeit, dass in einer Gruppe von 23 Personen mindestens zwei am gleichen Tag Geburtstag haben.

Hinweis: P(mind. 2 gleich) = 1 - P(alle verschieden). P(alle verschieden bei n Personen) = 365/365 * 364/365 * 363/365 * ... * (365-n+1)/365

AUFGABE
# Starter Code
n_people = 23
p_all_different = 1.0

# Deine Lösung hier
✅ Musterlösung
LÖSUNG
n = 23  # Anzahl Personen im Raum

# Wir berechnen die Gegenwahrscheinlichkeit:
# "Alle haben verschiedene Geburtstage"
prob_no_shared = 1.0

for i in range(n):
    # Person i+1 muss einen anderen Tag als alle vorherigen haben.
    # Von 365 Tagen sind i bereits vergeben → (365-i)/365 Wahrscheinlichkeit
    prob_no_shared *= (365 - i) / 365

# P(mind. 1 gemeinsam) = 1 - P(alle verschieden)
prob_shared = 1 - prob_no_shared
print(f"Wahrscheinlichkeit bei {n} Personen: {prob_shared:.1%}")
# Überraschend: Bei nur 23 Personen ist es bereits wahrscheinlicher DASS
# zwei den gleichen Geburtstag haben, als dass nicht!
A4

Wann bin ich halb so alt?

int · arithmetic · variables

Gegeben dein aktuelles Alter und das Alter einer anderen Person: In wie vielen Jahren wird Person A halb so alt sein wie Person B? Löse die Gleichung und berechne das Ergebnis.

AUFGABE
# Starter Code
age_a = 20
age_b = 50

# Wann gilt: age_a + x = (age_b + x) / 2
# Deine Lösung hier
✅ Musterlösung
LÖSUNG
my_age = 25
parent_age = 50  # Elternteil ist aktuell doppelt so alt

# Der Altersunterschied bleibt immer konstant (beide werden gleich älter)
age_diff = parent_age - my_age

# Wann war ich halb so alt wie mein Elternteil?
# Gesucht: my_age_then, sodass parent_age_then = 2 * my_age_then
# parent_age_then = my_age_then + age_diff
# → my_age_then + age_diff = 2 * my_age_then
# → age_diff = my_age_then
my_age_then = age_diff
years_ago = my_age - my_age_then

print(f"Vor {years_ago} Jahren war ich {my_age_then} und mein Elternteil {my_age_then + age_diff}")

list · append · remove · slicing

Erstelle eine leere Einkaufsliste. Füge 5 Artikel hinzu, entferne einen, füge am Anfang einen neuen ein, und gib die letzten 3 Artikel aus.

AUFGABE
# Starter Code
shopping_list = []

# Füge 5 Artikel hinzu (append)
# Entferne einen Artikel (remove oder pop)
# Füge am Anfang einen Artikel ein (insert)
# Gib die letzten 3 Artikel aus (slicing)
✅ Musterlösung
LÖSUNG

print(f"Anzahl Artikel: {len(einkaufszettel)}")

A7

Analyzing Cardiac Cycles

list · slicing · indexing · min · max

Gegeben ist eine Liste mit einem EKG-Zyklus. Bestimme den Minimal- und Maximalwert, schneide den positiven Peak-Bereich heraus (Index 3 bis 8) und berechne den Mittelwert dieses Ausschnitts.

AUFGABE
# Starter Code
one_cycle = [-99, 60, 62, 64, 68, 150, 200, 180, 120, 80, 60, 40, 20, -10, -40, -80, -99]

# 1. Min- und Maxwert bestimmen
# 2. Peak-Bereich herausschneiden (Index 3 bis 8)
# 3. Mittelwert des Peaks berechnen (ohne import: sum() / len())
✅ Musterlösung
LÖSUNG
one_cycle = [-99, 60, 62, 64, 68, 75, 90, 110, 130, 150, 170,
             185, 175, 155, 130, 105, 80, 65, 60, 58, 55, -99]

# -99 ist ein Sentinel-Wert (Markierung), kein echter EKG-Messwert
# List Comprehension filtert -99 heraus — nur echte Messwerte behalten
valid = [v for v in one_cycle if v != -99]

print(f"Anzahl Samples: {len(valid)}")
print(f"Min: {min(valid)}, Max: {max(valid)}")

# sum() / len() ist der manuelle Weg für den Mittelwert (ohne numpy)
print(f"Mittelwert: {sum(valid)/len(valid):.1f}")

A8

Cardiac Cycles Dictionary

dict · list · indexing · keys

Lege ein Dictionary an, das mehrere EKG-Zyklen speichert. Greife auf einzelne Zyklen zu, füge einen neuen Zyklus hinzu und iteriere über alle Keys.

AUFGABE
# Starter Code
cycles_dict = {
    "cycle_1": [-99, 60, 62, 64, 68, 150, 200, 180, 120, 80, 60, 40, 20, -10, -40, -80, -99],
    "cycle_2": [-95, 55, 60, 66, 70, 145, 195, 175, 115, 75, 55, 35, 15, -15, -45, -85, -95],
    "cycle_3": [-100, 58, 63, 65, 69, 148, 198, 178, 118, 78, 58, 38, 18, -12, -42, -82, -100],
}

# 1. Greife auf cycle_2 zu und gib seinen Maximalwert aus
# 2. Füge einen neuen Zyklus "cycle_4" hinzu (eigene Werte)
# 3. Iteriere über alle Keys und gib jeweils Länge des Zyklus aus
✅ Musterlösung
LÖSUNG
cycles_dict = {
    'patient_001': [-99, 60, 75, 110, 150, 185, 175, 130, 80, 60, -99],
    'patient_002': [-99, 55, 70, 100, 145, 180, 170, 125, 75, 55, -99],
}

# .items() gibt uns Schlüssel-Wert-Paare — ideal für Dictionary-Iteration
for patient, cycle in cycles_dict.items():
    # Sentinel herausfiltern, dann Statistiken berechnen
    valid = [v for v in cycle if v != -99]
    print(f"{patient}: {len(valid)} Samples, Max={max(valid)}")

EXA 1

Planetare Waage

assignment · operations · comparisons · numpy

Wieviele Erden braucht man, um Jupiter zu wiegen? Berechne die Massen beider Planeten (Volumen × Dichte) und vergleiche mit Operatoren.

  • Erde: Durchmesser 12 756 km, Dichte 5,51 × 10²⁴ kg/km³
  • Jupiter: Durchmesser 142 984 km, Dichte 1,33 × 10²⁴ kg/km³
  • Volumen Kugel: V = (4/3) · π · r³
AUFGABE
import numpy as np

# Durchmesser (km)
diameter_earth   = 12756
diameter_jupiter = 142984

# Dichten (kg/km³)
density_earth   = 5.51e24
density_jupiter = 1.33e24

# Berechne Massen und Vergleiche
# Masse = Volumen * Dichte,  Volumen = (4/3) * pi * r^3
✅ Lösung
LÖSUNG
import numpy as np

diameter_earth   = 12756
diameter_jupiter = 142984
density_earth    = 5.51e24
density_jupiter  = 1.33e24

r_earth   = diameter_earth   / 2
r_jupiter = diameter_jupiter / 2

mass_earth   = (4/3) * np.pi * r_earth**3   * density_earth
mass_jupiter = (4/3) * np.pi * r_jupiter**3 * density_jupiter

# Wie viele Erden wiegen so viel wie Jupiter?
n = 1
while mass_earth * n < mass_jupiter:
    n += 1
print(f"Es braucht {n} Erden, um Jupiter zu wiegen.")
print(f"Zum Vergleich: mass_earth * {n-1} < mass_jupiter ist {mass_earth*(n-1) < mass_jupiter}")

EXA 4

Temperaturüberwachung

dict · list · indexing · slicing · sum · len

Ein Temperatursensor liefert für 3 Maschinenzustände Messreihen. Jeder zweite Wert ist unzuverlässig (Index 0::2 = verlässlich). Berechne den mittleren Temperaturunterschied zwischen den Zuständen.

AUFGABE
sensor_data_dict = {
    '1602914400000': {
        'machine_state': 'starting',
        'batch_data': [34, 36, 40, 41, 33, 35, 39, 42, 40, 41, 35, 39]
    },
    '1602921600000': {
        'machine_state': 'running',
        'batch_data': [55, 60, 54, 52, 50, 55, 53, 61, 60, 59, 56, 57]
    },
    '1602928800000': {
        'machine_state': 'stopping',
        'batch_data': [65, 67, 70, 72, 70, 65, 73, 71, 68, 69, 66, 67]
    }
}

# Berechne den Mittelwert jedes Zustands (nur jeder 2. Wert: [0::2])
# Gib die Differenzen zwischen den Zuständen aus
✅ Lösung
LÖSUNG
sensor_data_dict = {
    '1602914400000': {'machine_state': 'starting',  'batch_data': [34,36,40,41,33,35,39,42,40,41,35,39]},
    '1602921600000': {'machine_state': 'running',   'batch_data': [55,60,54,52,50,55,53,61,60,59,56,57]},
    '1602928800000': {'machine_state': 'stopping',  'batch_data': [65,67,70,72,70,65,73,71,68,69,66,67]},
}

means = {}
for key, val in sensor_data_dict.items():
    reliable = val['batch_data'][0::2]          # jeder 2. Wert
    mean     = sum(reliable) / len(reliable)
    means[val['machine_state']] = mean
    print(f"{val['machine_state']:10s}: Mittelwert = {mean:.1f} °C")

print(f"\nRunning - Starting  = {means['running']  - means['starting']:.1f} °C")
print(f"Stopping - Running  = {means['stopping'] - means['running']:.1f} °C")
