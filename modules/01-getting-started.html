🚀 Modul 01 · Setup & Tools

Setup & Tools

Anaconda, Virtual Environments, JupyterLab, Spyder, PyCharm und Google Colab — alles was du zum Starten brauchst.

🐍 Anaconda installieren

Für diesen Kurs nutzen wir Anaconda — eine fertige Python-Umgebung mit allen wichtigen Paketen (NumPy, Pandas, Matplotlib u.v.m.) bereits vorinstalliert.

💡
Was ist Anaconda?

Anaconda ist eine Open-Source-Distribution für Python (und R). Sie vereinfacht die Installation und Nutzung von Tools und Packages erheblich — ein „Single Point of Truth" für alle Python-Aktivitäten.

Schritt-für-Schritt Installation

  1. Download: anaconda.com/products/individual → Version für dein Betriebssystem wählen
  2. Installer starten: Setup-Assistenten folgen, Standard-Optionen belassen
  3. Prüfen: Anaconda Prompt öffnen und eingeben:
BASH 
python --version
# Erwartet: Python 3.x.x

conda --version
# Erwartet: conda 23.x.x

🔒 Virtual Environments

Ein Virtual Environment (VE) ist eine abgeschlossene, isolierte Python-Umgebung. Es macht Sinn, VEs projekt- oder aufgabenbasiert zu erzeugen — so gibt es kein Chaos zwischen verschiedenen Projekten.

✅
Warum Virtual Environments?

Verschiedene Projekte brauchen manchmal verschiedene Package-Versionen. Ein VE hält diese sauber getrennt — Änderungen in einem Projekt beeinflussen keine anderen.

VE mit Anaconda Navigator erstellen

  1. Anaconda Navigator öffnen
  2. Links auf „Environments" klicken
  3. „Create" → Name: data-science-mit-python → Python-Version wählen → OK
  4. Das VE ist nach Erzeugung aktiv (grüner Pfeil neben VE)
  5. In diesem VE jetzt Spyder und JupyterLab installieren

Packages im VE über das Terminal installieren

Das Anaconda Prompt (Terminal) liegt dem VE zugrunde. Hierüber können Python-Erweiterungen – sogenannte Packages – mittels pip installiert werden:

BASH 
# VE aktivieren (falls nicht schon aktiv):
conda activate data-science-mit-python

# Einzelnes Package installieren:
pip install numpy

# Mehrere Packages auf einmal:
pip install pandas matplotlib seaborn plotly scikit-learn

📓 JupyterLab & Jupyter Notebooks

📔
Jupyter Notebook

Die klassische webbasierte Umgebung für interaktive Notebooks. Einfache, übersichtliche Oberfläche.

🧪
JupyterLab

Die neuere, leistungsfähigere Umgebung — wir nutzen hauptsächlich JupyterLab. Mehr Features, gleiche Notebooks.

Ein Jupyter-Notebook-Dokument ist ein JSON-Dokument mit einer Liste von Ein- und Ausgabezellen, die Code, Text und Plots enthalten können.

JupyterLab starten

BASH 
# Im Anaconda Prompt:
jupyter lab

# Öffnet den Browser automatisch unter:
# http://localhost:8888

Das Interface

Nach dem Start sehen wir den Launcher. Hier können neue Notebooks erzeugt werden. Die wichtigsten Interface-Elemente:

Element Funktion
LauncherStartseite — neues Notebook erstellen
File BrowserOrdner und Dateien ansehen, Notebooks öffnen
Run Cell ▶Aktuelle Zelle ausführen
Run Notebook ⏭Alle Zellen der Reihe nach ausführen
Stop ⏹Ausführung abbrechen
Restart Kernel 🔄Python-Prozess neu starten, alle Variablen löschen

Die drei Zell-Typen

💻 Code

Python-Code wird hier geschrieben und ist direkt ausführbar. Der Output erscheint direkt darunter.

📝 Markdown

Text, Überschriften, Listen, Formeln (LaTeX). Zum Dokumentieren und Strukturieren des Notebooks.

📄 Raw

Roher Text ohne Formatierung. Wird kaum benötigt.

👋 Das erste Programm

Das klassische erste Programm in Python — Hello World!:

PYTHON 
message = "Hello World!"
print(message)
Output
Hello World!

Was passiert hier?

  • message — eine Variable: ein benannter Speicherplatz für einen Wert
  • "Hello World!" — ein String (Text), erkennbar an den Anführungszeichen
  • print() — eine eingebaute Funktion, die etwas auf dem Bildschirm ausgibt

⚙️ Kernel & Ausführungsreihenfolge

⚠️
Wichtig: Reihenfolge der Zellen

Jupyter merkt sich alle Variablen im Kernel. Wenn du Zellen in falscher Reihenfolge ausführst, kann es zu Fehlern kommen. Bei Problemen: Kernel → Restart & Run All.

PYTHON 
# Fehler: Variable heißt "mesage" statt "message" (Tippfehler)
message = "Hello World!"
print(mesage)   # NameError!
Fehler
NameError: name 'mesage' is not defined
🔴
NameError

Einer der häufigsten Fehler! Python kennt die Variable nicht — meistens ein Tippfehler. Python unterscheidet Groß-/Kleinschreibung: message ≠ Message.

💬 Kommentare

Kommentare erklären den Code — sie werden von Python ignoriert.

PYTHON 
# Einzeiliger Kommentar mit dem #-Zeichen
my_variable = 5

def my_function(x):
    '''
    Mehrzeiliger Kommentar (Docstring).
    Wird für Funktionsbeschreibungen genutzt.
    '''
    return x ** 2

📁 Dateitypen: .py und .ipynb

📔 .ipynb

Jupyter Notebook Dateien. Gespeichert im JSON-Format — aufgebaut wie ein Dictionary in Python.

  • Enthält Code, Outputs und Text gemischt
  • Von normalen IDEs oft nicht direkt lesbar
  • Hauptformat für diesen Kurs
🐍 .py

Python-Skript-Dateien. Die originäre Dateiendung für Python-Programme.

  • Reiner Python-Code, kein Output gespeichert
  • Von allen Python-IDEs lesbar
  • Für größere Software-Projekte (z.B. in PyCharm)

🛠️ Weitere Tools: Spyder, PyCharm, Google Colab

Neben JupyterLab gibt es weitere Tools — wichtig zu wissen, wann man welches nutzt:

📌
Empfehlung des Professors

Wir nutzen hauptsächlich JupyterLab. Für größere Programme steigen wir auf PyCharm um. Spyder kann als „zur Vollständigkeit erwähnt" betrachtet werden.

🔬 Spyder

Wissenschaftliche Python-IDE. Ähnlich wie MATLAB. Gut für direktes Debugging. Wird in diesem Kurs nur selten genutzt.

⌨️ PyCharm

Professionelle IDE für Python-Softwareentwicklung. Ideal wenn längere Programme geschrieben werden (z.B. Webapps, APIs).

☁️ Google Colab

Jupyter Notebooks direkt im Browser — kostenlose Rechenressourcen. Fallback-Lösung wenn die lokale Installation Probleme macht.

Was für was?

Aufgabe Tool
Datenanalyse, Exploration, VisualisierungJupyterLab ← Haupttool im Kurs
Größere Python-Programme / SoftwareentwicklungPyCharm
Direktes Debugging, Scientific ComputingSpyder
Keine lokale Installation möglichGoogle Colab (Fallback)
